## 2026-06-03 调试控制台特殊豁免补丁开发 (当前进展)
- **发现问题**：当前台测试模式（菜单 5）启动时，由于用户正在看着该管理控制台窗口，当前的最前台活动窗口是助手自身的命令行窗口，不属于 VS Code/反重力白名单标题。导致安全约束机制会在此调试状态下无情拦截本来合法的蓝色按钮（输出 `非前台活动开发窗口的疑似色块` 拦截日志，无法点下按钮）。
- **重构方案**：
  - **调试窗标题豁免（Console Window Exemption）**：在检测中加入本管理器名称判定（如 `auto submitter manager`）。如果最前台窗口是控制台自身，说明用户正在进行前台测试，对此特殊状态实施安全包容，放行对全屏（除系统任务栏）合法按钮的保底点击。
- **已部署并推送**：
  - 本地及桌面文件已部署完成。
  - Git 已完成 commit 并成功推送到 GitHub 远程仓库中。
