## 2026-06-03 Win32 原生窗口枚举与兜底匹配机制 (当前进展)
- **发现问题**：用户生成的诊断报告表明 `检测到的开发窗口白名单数量: 0`。这是因为在管理员权限下，第三方 Python 库 `pygetwindow` 会由于进程安全隔离，无法正确读取或匹配普通用户权限的窗口标题（返回为空），从而导致 `is_point_in_windows` 校验始终失败，即使识别出蓝色按钮也无法触发点击。
- **重构方案**：
  1. **Win32 原生 EnumWindows API 替换**：弃用第三方 pygetwindow 库，在 `auto_submitter.py` 中完全改用 ctypes 声明 Windows 原生的 `EnumWindows` 和 `GetWindowTextW` 函数。原生 Win32 API 可以在管理员身份下稳定遍历绝大多数窗口的几何矩形与标题信息。
  2. **智能兜底（Fallback）点击机制**：在主循环中，如果未能检索到任何白名单窗口（即白名单数量为 0，大概率是系统会话隔离），但全屏范围内确实检测到了符合 Submit/Retry 几何外形和蓝色 HSV 的有效按钮，脚本不会直接放弃，而是自动切换至保底策略，对该按钮执行点击与回车。
- **下一步**：引导用户进行测试，体验原生 API 遍历与安全兜底机制下的点击成效。
