  - **空标题放行机制（Empty Active Title Exemption）**：修复了在 Hidden 隐藏窗口静默后台运行模式下，Python 进程因 Windows 安全隔离无法正确调用 `GetForegroundWindow` 读取普通前台窗口标题（返回为空标题），进而被错误当成无关窗口安全拦截导致“打印了成功但实际没点击”的 Bug。当获取前台窗口标题为空时，予以放行保底，只通过最底 75 像素任务栏规避，保证后台高可用性。
  - **0.50 秒重检延时**：将后台静默发送点击后到重新检测之间的 `time.sleep()` 延迟从 `0.35` 秒提升到 `0.50` 秒，避开 Windows 截屏缓冲带来的未点击误判。

## 2026-06-03 后台静默误判修复（局部蓝色像素比对校验） (当前进展)
- **背景与原因分析**：
  - 在控制台窗口最小化或完全被遮挡时，当执行 `background_click` 发送后台点击消息后，虽然 Electron 架构的目标窗口（VS Code）在后台忽略了该点击（即弹窗并未真正消掉），但它渲染了 Hover 或 Active 态，导致按钮外观发生细微的颜色/几何变化（如宽高比或面积微调）。
  - 这导致 0.5 秒后的 `recheck_btns = find_blue_buttons()` 因为几何过滤器校验或颜色超限直接漏检（返回空），从而被误判为“按钮已点消，后台静默确认已成功完成”，直接跳过了物理激活与物理 Enter 的点击兜底。
- **已完成局部蓝色像素二次校验（Local Blue Check）**：
  - **局部色彩兜底判定**：在 `execute_click_with_fallback` 阶段引入 `check_local_blue_exists`。当全局未匹配到合法几何特征的按钮时，直接截取原按钮所在的 `(w + 20) x (h + 20)` 像素局部区域，并统计其中符合 HSV 蓝色区间的总像素数。若大于 `50` 像素（弹窗未消失时必然存在大量蓝色像素），则判定按钮依然存在。
  - **异常回退策略**：若全局或局部重新检测中抛出截屏异常（可能由于 GDI 资源短暂受阻），程序默认保守认为按钮“依然存在”，保证物理兜底物理 Enter 的百分百可靠触发。
- **已部署并推送**：
  - 本地工作区 `auto_submitter.py` 和桌面管理脚本均同步更新完成。
