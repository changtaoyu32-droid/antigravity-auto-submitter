## 2026-06-03 后台静默误判修复（局部蓝色像素比对校验）
- **背景与原因分析**：
  - 在控制台窗口最小化或完全被遮挡时，当执行 `background_click` 发送后台点击消息后，虽然 Electron 架构的目标窗口（VS Code）在后台忽略了该点击（即弹窗并未真正消掉），但它渲染了 Hover 或 Active 态，导致按钮外观发生细微的颜色/几何变化（如宽高比或面积微调）。
  - 这导致 0.5 秒后的 `recheck_btns = find_blue_buttons()` 因为几何过滤器校验或颜色超限直接漏检（返回空），从而被误判为“按钮已点消，后台静默确认已成功完成”，直接跳过了物理激活与物理 Enter 的点击兜底。
- **已完成局部蓝色像素二次校验（Local Blue Check）**：
  - **局部色彩兜底判定**：在 `execute_click_with_fallback` 阶段引入 `check_local_blue_exists`。当全局未匹配到合法几何特征 of 按钮时，分析该区域内是否依然存在符合 HSV 亮蓝色范围的像素。

## 2026-06-03 双重加固补丁（高 DPI 局部脱靶与 CMD 窗口挂起冻结） (当前进展)
- **已修复高 DPI 局部截图脱靶 Bug**：
  - **原因**：`pyautogui.screenshot(region=...)` 在 Windows 开启 DPI 缩放（如 125%/150%）时，Pillow 底层对坐标进行了缩放处理，使得截取的局部图与设想的位置产生严重偏折脱靶，返回蓝色像素为 0，导致 `local_blue` 判定失败。
  - **修复**：彻底废弃 PyAutoGUI `region` 传参，改用全屏物理截图后直接通过 **NumPy 切片 (`img[top:bottom, left:right]`)** 提取局部图像，确保 100% 物理像素精确对齐，绝无偏移。
- **已修复控制台快速编辑模式 (QuickEdit) 线程挂起 Bug**：
  - **原因**：Windows CMD 命令提示符默认开启 QuickEdit。当控制台失去焦点、或被无意点中时，控制台会将 Python 的 I/O 线程**彻底挂起（Freeze）**。此时 Python 不会执行任何点击操作。一旦用户手动点击控制台展示并激活，命令行解除挂起，程序才瞬间解冻继续。
  - **修复**：在 `__main__` 初始化阶段使用 Win32 原生 API 彻底禁用控制台的 **QuickEdit Mode**，防止小助手在后台运行时被 Windows 自动挂起。
- **已部署并推送**：
  - 本地及桌面版 `auto_submitter.py` 已全部更新同步，并推送至 GitHub 远程公开仓库。
